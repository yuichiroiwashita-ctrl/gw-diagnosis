# gw-diagnosis — 業務効率化 診断フォーム

商談時に導入効果測定を提示するための診断フォーム。ヒアリング内容を入力すると診断結果・コスト試算・AI提案文を即時生成する。

## URL

http://webapps.insp.co.jp/gw-diagnosis/

## 概要

- 静的 HTML 単一ファイル（`index.html`）— GitHub Pages でホスティング
- 診断データは Supabase（PostgreSQL）に保存、複数端末で履歴を共有可能
- 外部依存：Chart.js 4.4.0 / Supabase JS Client 2（いずれも CDN）

## インフラ構成

| 役割 | サービス |
|------|----------|
| フロントエンド | GitHub Pages |
| データベース | Supabase（プロジェクト：gw-diagnosis、東京リージョン） |

## フォーム構成

### Step 1 — 基本情報

| 項目 | 内容 |
|------|------|
| 施設名・会社名 | 必須 |
| 業種 | 必須（10業種から選択） |
| 住所 | 任意 |
| 電話番号 | 任意 |
| 職員数 | 人数（コスト試算に使用） |
| 月間稼働日数 | デフォルト 22 日 |
| 1人あたり時給目安 | デフォルト 1,500 円 |

### Step 2 — 業務課題チェック（Q1〜Q17）

各質問に選択肢 + 備考テキストエリア（任意）を表示。

| カテゴリ | 質問 |
|----------|------|
| 情報共有 | Q1 連絡手段の数 / Q2 連絡内容の検索性 / Q3 伝達漏れ / Q13 メール対応時間 |
| ファイル管理 | Q4 最新版管理 / Q5 個人PC・USB保存 / Q6 資料検索時間 |
| 写真・データ管理 | Q7 個人スマホ撮影 / Q8 PC転送作業 / Q9 対象者ごとの写真確認 |
| セキュリティ・引継ぎ | Q10 退職者引き継ぎ / Q11 個人アカウント業務利用 / Q12 業務の属人化 |
| 業務効率化 | Q14 お便り・資料作成 / Q15 AI活用意向 / Q16 紙・手書き業務 / Q17 外出先での業務対応 |

回答選択肢：`よくある（2点）` / `時々ある（1点）` / `ない（0点）`  
Q1のみ：`1つ（0点）` / `2〜3つ（1点）` / `4つ以上（2点）`

### Step 3 — 診断結果

| 要素 | 内容 |
|------|------|
| 課題スコア | 全17問の合計を100%換算（最大34点） |
| グレード | A（0〜24%）/ B（25〜49%）/ C（50〜74%）/ D（75%〜） |
| 月間ロスコスト | `(スコア% × 90分 ÷ 60) × 時給 × 職員数 × 稼働日数` で試算 |
| レーダーチャート | 5カテゴリのスコアを可視化 |
| カテゴリ別バー | 要対処 / 要改善 / 問題なし で色分け |
| AI診断レポート | 課題カテゴリ・AI活用意向に応じた提案文を動的生成 |
| ヒアリングメモ | 入力された備考を設問ごとにまとめて表示（未入力の場合は非表示） |
| 印刷 / PDF保存 | ブラウザの印刷機能を呼び出し |

## 過去の診断履歴

- 診断完了時に Supabase へ自動保存
- ヘッダーの「過去の診断」ボタンから一覧を閲覧
- 1件削除 / 全件削除に対応
- 複数端末・複数ブラウザで同じ履歴を参照可能

## データベース

**テーブル：`diagnoses`**

| カラム | 型 | 内容 |
|--------|----|------|
| id | bigint | 自動採番 |
| saved_at | timestamptz | 診断日時 |
| facility_name | text | 施設名・会社名 |
| industry | text | 業種 |
| address | text | 住所 |
| phone | text | 電話番号 |
| staff | int | 職員数 |
| work_days | int | 月間稼働日数 |
| wage | int | 時給目安 |
| answers | jsonb | 各設問の回答 |
| remarks | jsonb | 各設問の備考 |
| total_pct | int | 課題スコア（%） |
| grade | text | グレード（A〜D） |
| grade_desc | text | グレード説明文 |
| cat_scores | jsonb | カテゴリ別スコア |
| waste_mins | int | 1人あたり業務ロス（分/日） |
| monthly_loss | bigint | 月間ロスコスト（円） |

## 開発・更新フロー

1. `index.html` を編集
2. ブラウザで動作確認（入力〜結果表示〜履歴の一連フロー）
3. `git push` → GitHub Pages に自動反映
